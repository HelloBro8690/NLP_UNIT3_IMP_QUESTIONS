Parsing Techniques 6 Marks

Definition
Parsing is the process of analyzing a sentence to determine its grammatical structure according to a given grammar. A parser takes a string of words as input and produces a parse tree (or phrase structure tree) as output.
Types of Parsing

① Top-Down Parsing

  • Starts from the start symbol (S) and tries to derive the input sentence.
  • Expands grammar rules from the top until it reaches terminal (word-level) symbols.
  • Matches left-to-right against the input.
  • Strategy: S → NP VP → … → "Mary saw Bob"

Top-Down: Start symbol expands downward to match input words

  • Advantage: Goal-directed; only explores trees compatible with grammar.
  • Disadvantage: May re-expand rules (left-recursion causes infinite loops).

② Bottom-Up Parsing

  • Starts from the input words (leaves) and builds up to the start symbol S.
  • Reduces word sequences to grammar constituents.
  • Strategy: "Mary saw Bob" → NP V NP → NP VP → S

Bottom-Up: Words reduce upward to reach the start symbol S

  • Advantage: Never pursues parses not supported by actual input words.
  • Disadvantage: May build many constituents that never connect to S.

③ Left-Corner Parsing

  • A hybrid of Top-Down and Bottom-Up parsing.
  • Reads the leftmost word of a rule's RHS, then predicts the rest top-down.
  • For rule NP → Det N: sees "Mary" (Det), then predicts N top-down.

Left-Corner: Hybrid — recognize bottom-up, predict top-down

Conclusion: Top-down is goal-directed, bottom-up is data-directed, and left-corner combines both. Left-corner parsing is generally the most efficient for real NLP grammars as it avoids the pitfalls of the other two approaches.
Q2

Dependency Grammar 6 Marks

Definition
Dependency Grammar is a grammatical framework that describes the syntactic structure of a sentence in terms of binary asymmetric relations (dependencies) between words. One word (the head) governs another (the dependent/modifier).
Key Concepts
  • Every word except the root depends on exactly one other word.
  • The verb is typically the root/head of a sentence.
  • There are no phrase-structure (constituent) trees — only direct word-to-word links.
  • Each dependency is labeled (subject, object, modifier, etc.).
Working / Steps
  1. Identify the root word (usually the main verb).
  2. Find all words that directly depend on the root (subject, object, etc.).
  3. For each dependent, find its own dependents recursively.
  4. Label each dependency arc with its grammatical role.
Example — "Mary saw Bob"
Dependency Graph
saw ← ROOT / \ nsubj dobj / \ Mary Bob
  • saw → ROOT (head of sentence)
  • Mary → nsubj of "saw" (nominal subject)
  • Bob → dobj of "saw" (direct object)
Second Example — "Kim chased Lee"
chased ← ROOT / \ nsubj dobj / \ Kim Lee
Common Dependency Relations

Prepositional Modifierprepin ← park
Advantages
  • Language independence: Works well for free-word-order languages (e.g., Hindi, Russian).
  • Compact representation: No intermediate phrase nodes needed.
  • Direct word relations: Easier to extract semantic roles.
  • Simpler structure: Easier to annotate and parse in many NLP systems.
Conclusion: Dependency Grammar represents sentence structure as a network of head-dependent word pairs rather than nested phrase brackets. It is widely used in modern NLP parsers for tasks like information extraction and machine translation because of its simplicity and language-neutral design.

Grammar Development 6 Marks

Definition
Grammar Development in NLP is the systematic process of designing, writing, testing, and refining formal grammars (typically Context-Free Grammars) to correctly parse natural language sentences. It involves incrementally adding rules to cover more linguistic phenomena.
Stages of Grammar Development

→
Fix Errors

Iterative grammar development cycle

  1. Start with a minimal grammar: Define basic rules for simple sentences (S → NP VP).
  2. Test on sample sentences: Try parsing "Mary saw Bob", "She eats".
  3. Identify gaps: Find sentences the grammar fails to parse.
  4. Add new rules: E.g., add rules for questions, negation, passives.
  5. Handle agreement: Add morphological constraints (subject-verb agreement).
  6. Resolve ambiguity: Refine rules to avoid over-generation.
Example — Building a Simple Grammar
Step 1: Minimal Grammar 
S  → NP VP
NP → Noun | Det Noun
VP → Verb | Verb NP
Noun  → Mary | Bob | Kim | Lee
Verb  → saw | chased | eats
Det   → the | a
This handles: "Mary saw Bob", "Kim chased Lee", "She eats"
Step 2: Extended Grammar (adding pronouns, adjectives) 
NP  → Pro | Adj NP
Pro → she | he | they
Adj → big | small | old
Now handles: "She eats", "The big dog chased Bob"
Challenges in Grammar Development
  • Over-generation: Grammar accepts ungrammatical sentences.
  • Under-generation: Grammar rejects valid sentences.
  • Agreement errors: "*She eat" must be rejected (number mismatch).
  • Ambiguity: One sentence can produce multiple parse trees.
Importance
  • Forms the foundation for building robust NLP parsers.
  • Used in rule-based question answering and information extraction systems.
  • Ensures grammatically correct output in natural language generation.
Conclusion: Grammar Development is an iterative process of writing, testing, and refining formal rules. A well-developed grammar precisely captures the grammatical patterns of a language, enabling accurate parsing without over- or under-generation.

Grammatical Features 6 Marks

Definition
Grammatical Features are formal properties of words or phrases that encode linguistic information such as number, person, gender, tense, and case. Features are expressed as attribute-value pairs and are used to enforce grammatical agreement between sentence constituents.
Common Grammatical Features
FeatureValuesExample
Numbersg (singular), pl (plural)"she" → sg; "they" → pl
Person1st, 2nd, 3rd"I" → 1st; "she" → 3rd
Gendermasc, fem, neuter"he" → masc; "she" → fem
Tensepast, present, future"saw" → past; "sees" → present
Casenom, acc, gen"she" → nom; "her" → acc
AgreementagrSubject-verb must match number/person
Feature Structures

Each word is associated with a set of feature-value pairs called a feature structure (also called Attribute-Value Matrix / AVM):

she → [ PER: 3rd, NUM: sg, CASE: nom, GEN: fem ] eats → [ TENSE: pres, AGR: [ PER: 3rd, NUM: sg ] ]
Agreement Using Features

Features enforce grammatical constraints. Subject and verb must agree in number and person:

Example
  • ✅ "She eats" — "she" is [3rd, sg]; "eats" requires [3rd, sg] → MATCH
  • ❌ "She eat" — "she" is [3rd, sg]; "eat" requires [1st/2nd or pl] → MISMATCH → Rejected
  • ✅ "Mary saw Bob" — "Mary" is [3rd, sg]; "saw" is past (no agreement needed in past)
How Agreement is Enforced
  1. Assign feature structures to all words in a lexicon.
  2. Grammar rules carry feature constraints.
  3. During parsing, features of daughter nodes are checked against parent node requirements.
  4. If features do not match → parse fails (sentence rejected).

Conclusion: Grammatical features encode linguistic properties as attribute-value pairs and are essential for enforcing agreement in NLP grammars. They are the building blocks of feature-based grammars and unification, enabling accurate parsing of natural language.
Q5

Unification in Feature-Based Grammar 6 Marks

Definition
Unification is an operation that combines two compatible feature structures into a single, merged feature structure. If the two structures share the same values for all overlapping features, unification succeeds; otherwise it fails. It is the core operation in feature-based grammar.
Unification Rules
  • Identical values: Two structures unify if their shared features have the same value.
  • Unspecified value (ANY): A variable or unspecified feature unifies with any value.
  • Conflict: If two structures disagree on a feature value → Unification FAILS.
  • Result: The unified structure contains all features from both, plus any resolved variables.
Unification Algorithm (Steps)
  1. Take two feature structures FS1 and FS2.
  2. For each feature in FS1, check if FS2 has the same feature.
  3. If both have the feature with the same value → keep it.
  4. If one has the feature and the other does not → add it to the result.
  5. If both have the feature with different values → FAIL.
  6. Return the merged structure.

Example — Failed Unification
Trying to unify "she" with "eat"
she: [ NUM: sg, PER: 3rd ] eat: [ AGR: [ NUM: pl ] ] ← expects plural Result: Unification FAILS ❌ sg ≠ pl → CONFLICT

❌ "She eat" is rejected — number mismatch.

Importance of Unification
  • Elegant way to enforce grammatical agreement without separate rules for each case.
  • Supports variable sharing — one feature value can be linked to multiple positions.
  • Foundation for powerful grammar formalisms like HPSG, LFG, and PATR-II.
  • Allows grammars to remain compact while covering many agreement phenomena.
Conclusion: Unification is the central mechanism of feature-based grammar. It merges feature structures when they are compatible and rejects combinations that violate grammatical constraints. This simple but powerful operation underpins agreement checking in modern NLP parsers.
Q6

Feature-Based Parsing 6 Marks

Definition
Feature-Based Parsing is a parsing approach that integrates grammatical features (attribute-value pairs) into grammar rules and uses unification to enforce agreement constraints during parsing. It extends standard CFG parsing to handle morphological and syntactic agreement.
How It Works

In standard CFG, rules like S → NP VP have no agreement check. Feature-based parsing augments rules with feature constraints:

Rule: S → NP[NUM:?n, PER:?p] VP[AGR:[NUM:?n, PER:?p]] Meaning: NP and VP must share the same NUM and PER values. ?n and ?p are variables — they unify to the same value.
Steps in Feature-Based Parsing
  1. Lexicon Lookup: Each input word is assigned a feature structure from the lexicon.
  2. Rule Application: Grammar rules are applied (top-down or bottom-up).
  3. Unification Check: At each rule application, feature structures of constituents are unified.
  4. Success: If unification succeeds → parse continues.
  5. Failure: If unification fails → that parse path is abandoned.
  6. Output: A parse tree annotated with feature structures at each node.

Complete Example — "She eats"
Step-by-Step Feature-Based Parse

Rejected Example — "She eat"
she → [ NUM: sg ] eat → [ AGR: [NUM: pl] ] Unification: sg ≠ pl → FAIL ❌ → Sentence Rejected
Advantages of Feature-Based Parsing
  • Agreement handling: Automatically enforces subject-verb, noun-adjective agreement.
  • Compact grammar: One rule with features replaces many specific rules.
  • Precise: Rejects ungrammatical sentences that CFG alone would accept.
  • Extensible: Easily extended with more features (case, gender, tense) without changing the parser.

Conclusion: Feature-Based Parsing combines the power of context-free parsing with morphological and syntactic agreement checking through unification. It is more accurate than plain CFG parsing and forms the basis of many real-world NLP grammar frameworks.
